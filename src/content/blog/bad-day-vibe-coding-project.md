## 건설 엔지니어가 왜 웹앱을 만들었을까

현장에서 하루를 보내고 나면, 말로 표현하기 어려운 감정이 남을 때가 있습니다. 공정이 꼬였을 때의 답답함, 준공 검사를 통과했을 때의 안도감. 그런 감정을 기록하고 싶었고, AI가 위로해주면 어떨까 하는 단순한 아이디어에서 **Bad Day** 프로젝트가 시작됐습니다.

> "나쁜 하루도 괜찮아." — Bad Day의 슬로건입니다.

![Bad Day 메인 화면 - 로그인 전](/blog/bad-day-hero.png)

## Bad Day가 하는 일

일기를 쓰면 AI가 감정을 분석하고, 그에 맞는 위로 메시지와 음악을 추천해주는 웹앱입니다.

- **AI 감정 분석** — Gemini API가 일기 내용에서 기쁨, 슬픔, 분노, 불안 등 7가지 감정을 분류합니다
- **맞춤 위로 메시지** — 감정에 맞는 개인화된 위로와 응원 메시지를 생성합니다
- **음악 추천** — 감정별로 큐레이션된 YouTube 음악을 자동으로 틀어줍니다
- **감정 대시보드** — 캘린더 히트맵으로 나의 감정 변화를 한눈에 볼 수 있습니다

## 실제 구동 화면

### 메인 화면 — 일기 작성과 최근 기록

로그인하면 바로 일기를 쓸 수 있는 에디터가 나타납니다. 아래에는 최근 일기 목록이 감정 이모지와 함께 표시됩니다. "오늘 하루는 어떠셨나요?"라는 질문에 자유롭게 적으면 됩니다.

![Bad Day 메인 화면 - 일기 작성 에디터와 최근 일기 목록](/blog/bad-day-main.png)

### 일기 상세 — AI 감정 분석 결과

일기를 제출하면 Gemini API가 감정을 분석합니다. 감정 종류와 강도(퍼센트)를 보여주고, **"AI의 한마디"**로 맞춤 위로 메시지를 생성합니다. 그 아래에는 감정에 어울리는 음악이 YouTube 플레이어로 바로 재생됩니다.

![일기 상세 화면 - 감정 분석, AI 위로 메시지, 음악 추천](/blog/bad-day-diary-detail.png)

"평온" 감정이 50%로 분석되었고, "오늘 하루도 수고했어요. 일기를 쓰는 것만으로도 대단한 거예요."라는 위로 메시지와 함께 Calm Ambient Piano가 추천되었습니다. 추가로 AI에게 더 이야기하거나, AI 액션 플랜을 만들 수도 있습니다.

### 일기장 — 감정의 타임라인

일기장 페이지에서는 모든 일기를 시간순으로 볼 수 있습니다. 각 일기에는 감정 이모지, AI 위로 메시지 미리보기, 날짜와 감정 태그가 표시됩니다. 한눈에 내 감정의 흐름을 파악할 수 있습니다.

![일기장 목록 - 감정별 이모지와 AI 메시지 미리보기](/blog/bad-day-diary-list.png)

### 감정 대시보드 — 나의 감정 지도

대시보드에서는 감정 통계를 한눈에 볼 수 있습니다. 총 일기 수, 감정 종류, 최다 감정이 상단에 표시되고, **감정 분포 차트**로 어떤 감정이 많았는지 비율로 확인할 수 있습니다.

![감정 대시보드 - 통계, 감정 분포, 캘린더 히트맵](/blog/bad-day-dashboard.png)

**감정 캘린더**는 날짜별로 그날의 대표 감정을 이모지로 보여줍니다. 색상으로 감정을 구분할 수 있어서, 한 달간 감정 변화의 패턴을 시각적으로 파악할 수 있습니다.

## 기술 스택 — 전부 무료로 구성했습니다

이 프로젝트의 기술 선택 기준은 하나였습니다. **월 운영비 0원.**

| 역할 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 15 | App Router + Server Actions로 백엔드 별도 구축 불필요 |
| AI | Gemini API | 무료 티어가 넉넉하고, 감정 분석 정확도가 높음 |
| DB | Supabase | PostgreSQL + RLS(Row Level Security) 무료 제공 |
| 인증 | Clerk | 소셜 로그인 포함 10K MAU 무료 |
| 배포 | Vercel | GitHub push만 하면 자동 배포 |
| UI | Shadcn UI + Tailwind CSS v4 | 다크 모드 기본, 일관된 디자인 시스템 |
| 상태관리 | Zustand + TanStack Query | 클라이언트/서버 상태 분리 |
| 음악 | YouTube Embed | API 키 없이 감정별 음악 큐레이션 |

**총 운영비: 월 $0입니다.** Gemini API, Supabase, Clerk, Vercel 모두 무료 티어 안에서 운영됩니다. 사이드 프로젝트에 돈을 쓸 이유가 없었습니다.

## 바이브코딩으로 만든 과정

프로그래밍 전공자가 아닌 건설 엔지니어로서, 이 프로젝트는 **AI와의 대화로 만들었습니다.**

### 피봇의 순간

처음에는 "실시간 상태 공유 보드(VibeBoard)"를 기획했지만, AI와 대화하면서 **감정 일기 앱으로 피봇**했습니다. "이 기능은 Supabase RLS로 이렇게 구현하면 돼"라고 AI가 알려주고, 저는 "사용자 경험을 이렇게 바꾸고 싶어"라고 방향을 잡았습니다.

### 개발 순서

1. **DB 스키마 + Gemini 연동** — 일기 테이블 설계, AI 감정 분석 파이프라인 구축
2. **Server Actions** — 일기 CRUD + AI 분석 로직을 서버 사이드로 구현
3. **UI 구현** — 메인, 일기 목록, 상세, 대시보드 페이지 제작
4. **음악 시스템** — 감정별 YouTube 음악 큐레이션 + 플레이어 통합
5. **디자인 폴리싱 + 배포** — 다크 모드 기반 UI 마무리, Vercel 배포

### 바이브코딩의 장점

1. **아이디어에서 프로토타입까지 빠릅니다** — DB 스키마 설계부터 UI까지 대화로 진행
2. **모르는 기술도 쓸 수 있습니다** — Clerk, Supabase RLS, Zustand, TanStack Query 등 처음 쓰는 도구도 AI가 가이드
3. **피봇이 쉽습니다** — VibeBoard에서 Bad Day로 방향 전환이 부담 없었음

### 주의할 점

반면 주의할 점도 있었습니다:

- AI가 생성한 코드의 **보안 설정은 직접 검증**해야 합니다 (특히 RLS 정책)
- 전체 아키텍처는 **사람이 이해하고 있어야** 디버깅이 가능합니다
- "동작은 하는데 왜 이렇게 되는지 모르겠다"는 상태를 피해야 합니다

실제로 Supabase RLS 정책에서 다른 사용자의 일기가 보이는 버그가 있었는데, AI가 생성한 정책을 그대로 쓴 것이 원인이었습니다. 이런 부분은 반드시 직접 검증해야 합니다.

## 배운 것

이 프로젝트에서 가장 크게 느낀 건, **바이브코딩은 "코딩을 대신해주는 것"이 아니라 "내가 만들 수 있는 범위를 넓혀주는 것"**이라는 점입니다.

건설 현장에서 문제를 파악하고 해결책을 설계하는 능력 — 그게 소프트웨어를 만들 때도 그대로 적용됩니다. AI는 구현을 도와주지만, **무엇을 만들지, 왜 만드는지**는 여전히 사람의 몫입니다.

건설 엔지니어가 웹앱을 만들 수 있다면, 여러분도 할 수 있습니다. 중요한 건 기술력이 아니라 **해결하고 싶은 문제가 있는지**입니다.
