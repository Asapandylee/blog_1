매주 금요일, 자재 정산 엑셀 파일을 열면 한숨부터 나왔습니다.

3개 현장에서 올라오는 자재 입출고 데이터를 하나의 양식으로 합치고, 단가를 매칭하고, 월간 리포트를 만드는 작업. 숙련자가 해도 **8시간**은 걸리는 반복 노동이었습니다.

"이걸 왜 사람이 하고 있지?"

그 질문 하나가 모든 걸 바꿨습니다.

## 문제 분석: 뭘 자동화할 수 있을까

먼저 매주 반복되는 작업을 쪼개봤습니다.

| 작업                     | 소요 시간 | 자동화 가능? |
| ------------------------ | --------- | ------------ |
| 3개 현장 엑셀 파일 병합  | 1시간     | ✅           |
| 자재 코드 ↔ 단가 매칭    | 2시간     | ✅           |
| 수량 검증 (이상치 체크)  | 1.5시간   | ✅           |
| 월간 리포트 양식 생성    | 2시간     | ✅           |
| 담당자 확인 및 메모 입력 | 1.5시간   | ❌           |

8시간 중 **6.5시간이 기계적 반복**이었습니다. 사람이 꼭 판단해야 하는 건 마지막 검토뿐이었죠.

## 해결: pandas 하나면 충분했다

거창한 시스템이 필요하지 않았습니다. Python의 `pandas` 라이브러리 하나로 대부분 해결됐습니다.

```python
import pandas as pd
from pathlib import Path

# 3개 현장 엑셀 파일을 하나로 병합
files = Path("./data/weekly").glob("*.xlsx")
merged = pd.concat([pd.read_excel(f) for f in files], ignore_index=True)

# 자재 코드-단가 매칭
price_table = pd.read_excel("./data/master/price_master.xlsx")
result = merged.merge(price_table, on="자재코드", how="left")

# 이상치 체크: 전주 대비 30% 이상 증감 플래그
result["전주대비"] = result.groupby("자재코드")["수량"].pct_change()
result["이상여부"] = result["전주대비"].abs() > 0.3
```

핵심은 이 코드를 처음부터 직접 짠 게 아니라, **AI에게 엑셀 양식을 보여주고 "이거 자동으로 합쳐줘"라고 요청**한 것입니다. 바이브코딩의 위력을 처음 실감한 순간이기도 합니다.

## 결과: 8시간 → 30분

자동화 적용 후 워크플로우가 이렇게 바뀌었습니다.

1. 각 현장에서 엑셀 파일을 폴더에 드롭
2. 스크립트 실행 (10초)
3. 생성된 리포트를 열어 **검토만** 수행 (30분)

주 8시간이 30분으로 줄었습니다. **월 기준 약 30시간 절약**입니다.

## 교훈: 코드를 몰라도 자동화할 수 있다

이 경험에서 얻은 가장 큰 인사이트는 "코딩 실력이 없어도 자동화는 가능하다"는 것입니다. AI 코딩 도구가 발전한 지금, 필요한 건 프로그래밍 지식이 아니라 **"뭘 자동화해야 하는지 정확히 아는 것"**입니다.

건설 현장에는 이런 반복 작업이 아직 수없이 남아있습니다. 다음 타깃은 **일일 안전점검 리포트 자동 생성**입니다.
