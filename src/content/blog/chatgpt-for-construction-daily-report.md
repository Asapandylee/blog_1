건설 현장 일일 보고서. 매일 써야 하지만, 매일 귀찮은 그것.

오늘 인력 투입 현황, 주요 작업 내용, 안전 이슈, 내일 계획. 내용 자체는 머릿속에 다 있는데, 이걸 양식에 맞춰 깔끔하게 정리하는 데 **매일 30~40분**이 소모됩니다. 현장 마감 후 지친 상태에서 말이죠.

이 시간을 **5분으로 줄여**봤습니다.

## 핵심: 프롬프트 템플릿 만들기

ChatGPT를 단순히 "보고서 써줘"라고 부려서는 쓸만한 결과를 얻기 어렵습니다. **현장 맥락을 담은 프롬프트 템플릿**을 한 번 잘 만들어두면, 이후에는 데이터만 바꿔 넣으면 됩니다.

```
당신은 건설 현장 공무 담당자입니다.
아래 정보를 바탕으로 일일 공사 보고서를 작성해주세요.

[프로젝트명]: ○○동 주상복합 신축공사
[공정 단계]: 골조 공사 (지상 12층 중 8층 진행 중)
[오늘 날짜]: 2026-02-05

[투입 인력]
- 철근공 12명
- 형틀목공 8명
- 콘크리트공 6명
- 타워크레인 운전원 2명

[주요 작업]
- 8층 슬래브 철근 배근 완료
- 7층 거푸집 해체 50%
- 콘크리트 타설 준비 (내일 예정)

[안전 이슈]
- 특이사항 없음

[기상]
- 맑음, -2℃~5℃

양식은 회사 표준 보고서 형태로 작성하되,
핵심 사항은 굵은 글씨로 강조해주세요.
```

## 결과물의 품질

이 프롬프트로 생성된 보고서는 바로 제출할 수 있는 건 아닙니다. 하지만 **초안으로는 충분**합니다. 실제로 수정이 필요한 부분은 이 정도입니다.

| 항목             | 수정 필요도         |
| ---------------- | ------------------- |
| 전체 구조와 양식 | 거의 완벽           |
| 인력/수량 수치   | 검증만 하면 됨      |
| 작업 상세 내용   | 현장 특이사항 추가  |
| 안전 관련 문구   | 법적 표현 확인 필요 |

**초안 생성 → 5분 검토 및 수정 → 제출**. 이 루틴이 정착되면 매일 25분 이상을 아낄 수 있습니다.

## 더 나아가기: 음성 입력 연동

가장 이상적인 워크플로우는 퇴근 전 **현장을 걸으며 음성으로 오늘 내용을 말하고, AI가 보고서를 자동 생성**하는 것입니다. 이미 Whisper API와 ChatGPT를 연동하면 기술적으로 가능합니다.

현재 이 워크플로우를 간단한 웹앱으로 만들어보고 있는데, 완성되면 이 블로그에서 공유하겠습니다.

## 주의할 점

AI 생성 보고서를 사용할 때 반드시 기억해야 할 것이 있습니다.

- **수치는 반드시 직접 확인하세요.** AI는 그럴듯한 숫자를 지어낼 수 있습니다
- **안전 관련 내용은 더욱 신중하게.** 법적 책임이 따르는 부분은 꼭 검토하세요
- **회사 보안 정책을 확인하세요.** 프로젝트 정보를 외부 AI에 입력해도 되는지 사전 확인이 필수입니다

도구는 도구일 뿐, **최종 판단은 항상 엔지니어의 몫**입니다.
